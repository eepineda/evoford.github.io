Stores App - Mobile First

Use Case: Store staff quickly requests waste collection, tracks status, and views history.

The Stores app is designed with simplicity and speed in mind. Staff can log in quickly using store ID or by scanning a QR code. The home screen offers instant access to scan bin or waste-type QR codes, which auto-fills location and bin type on the request form. Users can enter volume, select a time window, and add notes.

After submitting, the request confirmation shows a scheduled status with estimated time of arrival (ETA). Staff can track their collection in real time as the status progresses through Requested, Accepted, In Transit, and Collected stages. A history section provides filters for date and waste type, helping users review past collections. Settings include support chat, FAQ, and profile management. Push notifications can keep users updated on their requests (especially if the app is installed as PWA or native).

Main Screens (5–7 screens):

  • Login / Sign-Up (fast access, store ID or QR onboarding)
  • Scan QR Code (Home Screen)
  • Request Collection Form (auto-filled location/bin type, volume, time window, notes)
  • Request Confirmation (status = Scheduled, ETA)
  • Track Collection (Requested → Accepted → In Transit → Collected)
  • History with filters
  • Settings / Help (support chat, FAQ, profile)

Business Management - Admin Portal

Designed for: Operators and planners managing routes and collections efficiently.

This desktop-first portal gives real-time KPIs, pending requests, and system alerts at login. Users can review and manage collection requests: approve, decline, or reschedule them easily. The schedule planning screen uses calendar views with advanced route optimization.

Driver assignment is simplified with drag-and-drop or dropdown selections, while the live map provides a clear overview of driver locations and active collections. Performance metrics track time efficiency and pickup completion rates. Client management helps maintain contact details and track collection history by store. Settings include user management and support tools.

Main Screens (6–8 screens):

  • Login / Admin Dashboard (KPIs, requests, alerts)
  • Collection Requests Overview (approve, decline, reschedule)
  • Schedule Planning (calendar, route optimization)
  • Driver Assignment Panel (drag/drop or dropdown)
  • Live Map View (driver locations, active collections)
  • Driver Performance (time efficiency, pickups)
  • Client Management (contacts, collection history)
  • Settings / Support / User Management

Drivers App - Mobile Experience

Designed for: Quick, mobile-only use by drivers.

The Drivers app provides an easy-to-use interface for daily tasks. Drivers log in to start their day, view an optimized route with map and list, and access detailed info for each collection task including address, waste type, and notes.

Status updates can be quickly set (Arrived, Collected, Issue Reported). GPS navigation is integrated for real-world route guidance or simulated deep links. At the end of the day, drivers get a summary of completed pickups, notes, and performance metrics.

Main Screens (4–6 screens):

  • Login / Daily Start
  • Today’s Route Overview (map + list)
  • Collection Task Detail (address, waste type, notes)
  • Update Status (Arrived, Collected, Issue Reported)
  • Map Navigation (GPS integration or simulation)
  • End of Day Summary (completed pickups, notes, metrics)
