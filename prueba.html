EvoSustain App Modules Overview

1. Stores App (Mobile-First)

Use case: Store staff quickly requests collection, tracks it, and views history.

Main Screens (5–7):

  • Login / Sign-Up: Fast access, possibly via store ID or QR onboarding.
  • Scan QR Code (Home Screen): Opens camera to scan bin/waste-type QR code.
  • Request Collection Form: Auto-fills location and bin type. Fields: volume, time window, optional notes.
  • Request Confirmation: Status = “Scheduled” with collection ETA.
  • Track Collection: Simple status updates: Requested → Accepted → In Transit → Collected.
  • History: List of past collections with filters by date/type.
  • Settings / Help: Support chat, FAQ, profile management.
  • Optional: Push notifications for collection updates.

2. Business Management (Admin Portal)

Designed for: Operators and planners. Desktop-first with possible map interface.

Main Screens (6–8):

  • Login / Admin Dashboard: KPIs, pending requests, system alerts.
  • Collection Requests Overview: Approve, decline, reschedule requests.
  • Schedule Planning: Calendar + route optimization logic.
  • Driver Assignment Panel: Drag-and-drop or dropdown assignment.
  • Live Map View: Real-time driver locations, active collections.
  • Driver Performance: Metrics: time efficiency, completed pickups.
  • Client Management: Contact info, collection history per store.
  • Settings / Support / User Management.

3. Drivers App (Mobile Experience)

Designed for: Quick daily use, mobile-only.

Main Screens (4–6):

  • Login / Daily Start: Driver login and workday start.
  • Today’s Route Overview: Map + list view, optimized route.
  • Collection Task Detail: Address, type of waste, special notes.
  • Update Status: Options: Arrived, Collected, Issue Reported.
  • Map Navigation: Integration with GPS or deep link simulation.
  • End of Day Summary: Completed collections, notes, metrics.
